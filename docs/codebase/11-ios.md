## Overview

SwiftUI app, iOS 17+ (`project.yml:4`), bundle `ai.transilience.cisocopilot`
(`project.yml:41`). It is a **read-mostly companion** to the AWS-native v2
backend: it authenticates via Cognito OAuth (ASWebAuthSession), then calls
API Gateway over HTTP with the Cognito **ID token** as Bearer. The app never
talks to upstream sources — only API Gateway — except the voice path, which
talks **directly to OpenAI Realtime** after the backend mints an ephemeral
key.

Three top-level states drive routing (`RootView.swift:13-45`): signed-out →
`SignInView`; signed-in-but-pending → `PendingApprovalView`; approved →
`MainTabView` (6 tabs). A push-notification tap layers a modal `BriefingView`
voice session over whatever is on screen (`CISOCopilotApp.swift:19-29`).

Single WebRTC SPM dependency: `stasel/WebRTC` from 130.0.0 (`project.yml:9-12,
32-34`), used only by `VoiceClient`.

**Vision note (from CLAUDE.md / MEMORY):** iOS is being rethought as a
lightweight **alerting + team-handoff companion**, NOT a web-feature port.
Several shipped screens (Risk Register CRUD, full findings rollup browser) read
as over-ported web features against that vision — flagged in
"Over-ported web features" below.

## b) Services Inventory (`ios/CISOCopilot/Services/`)

### `Services/APIClient.swift`
- **Purpose:** the single HTTP client for the v2 backend. `@Observable`
  (`APIClient.swift:7`). Holds a weak `AuthManager` (`:21`, bound in
  `CISOCopilotApp.onAppear`).
- **Base URL:** read from Info.plist key `API_BASE_URL`; `fatalError` if
  missing (`APIClient.swift:8-16`) → must copy `Local.xcconfig.example`.
- **Auth handling:** every authed call goes through `attachAuthHeader`
  (`:467-473`), which calls `auth.validIdToken()` and sets
  `Authorization: Bearer <id_token>`. Throws `APIError.notAuthenticated` if
  no token. `discoverTenant` (`:118`) is the **only unauthed** call.
- **Endpoints called** (METHOD path → method name, line):
  - `GET /me` → `fetchMe` (`:29`)
  - `POST /me/device-token` → `registerDeviceToken` (`:41`) — APNs token
  - `POST /tools/{name}` → `callTool` (`:52`) — voice tool dispatcher
  - `GET /connections` → `listConnections` (`:65`)
  - `POST /onboarding/{aws,azure,entra,gcp}/initiate` → `initiate*Onboarding` (`:74,84,94,104`)
  - `POST /auth/discover-tenant` → `discoverTenant` (`:118`, unauthed)
  - `POST /voice/session` → `voiceSession` (`:142`)
  - `GET /findings` → `listFindings` (`:151`), `findingsTotal` (`:171`), `listFindingsByCheck` (`:398`)
  - `GET /events` → `listEvents` (`:191`), `eventsTotal` (`:209`)
  - `GET/PATCH /policies` (+ `/enrich`, `/generate-all`) → `listPolicies`/`getPolicy`/`updatePolicy`/`enrichPolicy`/`generateAllPolicies` (`:229-278`) — **NOT called by any View**
  - `GET/PATCH /questionnaires` → `listQuestionnaires`/`getQuestionnaire`/`patchQuestionnaireItem`/`suggestQuestionnaireItem` (`:282-313`) — **NOT called by any View**
  - `GET/PUT /trust` → `getTrustPage`/`putTrustPage` (`:317-332`) — **NOT called by any View**
  - `GET/POST/PATCH /risks` → `listRisks`/`createRisk`/`updateRiskStatus` (`:336-376`)
  - `GET /findings/rollup` → `findingsRollup` (`:380`)
  - `GET /findings/summary` → `findingsSummary` (`:416`)
  - `GET /compliance/summary` → `complianceSummary` (`:426`)
  - `GET /entities`, `GET /entities/{id}` → `listEntities`/`getEntity` (`:436,450`)
- **DTOs:** all in this file (`:489-902`). See "Finding decoder contract".
- **Errors:** `APIError` (`:882`) — `badResponse`, `badStatus(Int)`,
  `notAuthenticated`, `discoveryFailed(Int,String)` (the last parses the
  server `message`/`error` body).

### `Services/AuthManager.swift`
- **Purpose:** Cognito OAuth 2.0 authorization-code flow w/ refresh.
  `@Observable`, `NSObject` conforming to
  `ASWebAuthenticationPresentationContextProviding` (`AuthManager.swift:13`).
- **Cognito config hardcoded** (`:17-48`): domain
  `ciso-copilot.auth.us-east-1.amazoncognito.com`, clientId
  `2r71e13kahf79bvb9stuehm3il`, scheme `cisocopilot`, redirect
  `cisocopilot://auth/callback`, scope `openid email profile`. **Not** from
  config — baked into the binary.
- **Flow:** `signIn(authorizeURL:)` (`:75`) opens `ASWebAuthenticationSession`
  (`:120-153`) with the discovered per-tenant authorize URL, captures the
  `code`, exchanges via `POST /oauth2/token` (`:170-186`), persists tokens.
  `prefersEphemeralWebBrowserSession=false` (`:150`) so SSO cookies persist.
- **Token model:** stores `id_token`, `access_token`, `refresh_token`,
  `id_token_expires_at` in Keychain. **`access_token` is stored but unused** —
  API Gateway's CognitoUserPoolsAuthorizer accepts **ID tokens**, so
  `validIdToken()` (`:101`) returns the ID token as the API JWT. Refreshes
  lazily when within 60s of expiry (`:104-115`).
- **Sign out** (`:89`): clears Keychain + best-effort Cognito `/logout` in the
  system browser.
- **Surfaces IdP errors** from the callback (`:139-143`) instead of dropping
  them.

### `Services/Keychain.swift`
- **Purpose:** minimal `kSecClassGenericPassword` wrapper, service
  `ai.transilience.cisocopilot` (`Keychain.swift:7`). Keys enum (`:9-14`):
  `id_token`, `access_token`, `refresh_token`, `id_token_expires_at`.
  Accessibility `kSecAttrAccessibleAfterFirstUnlock` (`:25`). `save` deletes
  then adds (`:23-26`).

### `Services/IncidentRouter.swift`
- **Purpose:** `ObservableObject` holding the currently-active incident for
  push-tap navigation (`IncidentRouter.swift:9`). Observes
  `.navigateToBriefing` (`:13-19`), builds `IncidentContext(findingId:payload:)`
  on the main thread (`:21-27`). `IncidentContext` equality is by `findingId`
  only (`:38-40`).

### `Services/VoiceClient.swift`
- **Purpose:** WebRTC OpenAI Realtime client. See "Voice flow" below.

## c) Voice flow — WebRTC `RealtimeClient` (`Services/VoiceClient.swift`)

`@MainActor @Observable` `VoiceClient: NSObject` (`VoiceClient.swift:20-22`).
WebRTC is used **deliberately, not WebSocket** — the header comment
(`:6-19`) documents that iPhone speaker at conversation distance overwhelmed
iOS's standalone AEC under the prior WebSocket impl, producing "jumbled,
repeating" output as VAD fired on echo; WebRTC's Google AEC3 handles it
natively. (Matches CLAUDE.md rule; reference impl is
`shasta-ios-poc/.../RealtimeClient.swift`.)

GA flow (`:8-13`, `start()` `:79-113`):
1. `api.voiceSession()` → `POST /voice/session` mints an ephemeral
   `client_secret` (`:90-100`). Backend is **not** on the audio path.
2. `configureRtcAudioSession()` (`:122-155`): `.playAndRecord`,
   `.defaultToSpeaker`+Bluetooth, **mode `.videoChat`** (VPIO echo cancel,
   louder than `.voiceChat`), `overrideOutputAudioPort(.speaker)` AFTER
   activation, and re-asserts speaker on every `routeChangeNotification`
   (`:145-154`) because iOS reverts one-shot overrides.
3. `setupPeerConnection()` (`:157-187`): `iceServers=[]` (OpenAI is the SFU,
   no STUN/TURN), unifiedPlan, local mic audio track, data channel
   `"oai-events"`.
4. `sdpExchange()` (`:189-235`): create offer → `POST
   https://api.openai.com/v1/realtime/calls` with `Bearer <ephemeral>` +
   `Content-Type: application/sdp` → set remote answer.
5. Events arrive on the data channel; `handleEvent` (`:254-317`) maps Realtime
   event types to state + transcript accumulation (user + assistant partials,
   `input_audio_transcription.*`, `output_audio_transcript.*`).

**Seed message** (`start(seedDeveloperMessage:)` `:74-77`): `BriefingView`
passes a built prompt; `sendSeedIfPending` (`:327-340`) injects it as a
user-role conversation item + `response.create` once the channel opens, so
Shasta speaks the briefing first.

**Tool dispatch** (`dispatchTool` `:344-397`): two tools handled locally —
`get_top_risks` (→ `api.listFindings`) and `list_connected_clouds`
(→ `api.listConnections`). All other tool names forward to
`api.callTool` → `POST /tools/{name}` (`:407-410`) — covers wow-demo tools
(slack_dm, create_jira_ticket, revoke_oauth_grant, create_pr_with_bump,
tail_lambda_logs_for_pattern, run_forensic_scan) per the comment (`:377-382`).
Result is returned to OpenAI as `function_call_output` + `response.create`
(`:388-396`).

Two consumers instantiate their own `VoiceClient`: `VoiceChatView`
(manual mic, `:9`) and `BriefingView` (auto-start w/ seed, `:8`).

## d) Push / APNs flow

1. `AppDelegate.didFinishLaunching` (`CISOCopilotApp.swift:47-60`) sets the
   `UNUserNotificationCenter` delegate, requests `.alert/.badge/.sound`
   authorization, then `registerForRemoteNotifications()` on grant.
2. `didRegisterForRemoteNotificationsWithDeviceToken` (`:62-69`) hex-encodes
   the token and posts `Notification.Name.deviceTokenReady` with the hex as
   `object`.
3. `RootView.onReceive(.deviceTokenReady)` (`RootView.swift:31-44`) — only if
   `auth.state==.signedIn` — calls `api.registerDeviceToken(hex)` →
   `POST /me/device-token` body `{"token": hex}` (`APIClient.swift:41-46`).
   The indirection exists because `AppDelegate` has no `@Environment` access
   to `APIClient` (documented `CISOCopilotApp.swift:40-43`).
4. **Tap handling** (`userNotificationCenter didReceive` `:77-93`): reads
   `userInfo["finding_id"]`, stringifies the whole payload, posts
   `.navigateToBriefing` (object=findingId, userInfo=payload). `IncidentRouter`
   turns that into `activeIncident`; `CISOCopilotApp` presents `BriefingView`
   as a sheet (`CISOCopilotApp.swift:19-29`).
5. Foreground push shows `[.banner, .sound]` (`:96-100`).

Entitlement `aps-environment: development` (`project.yml:38`).

## e) Finding decoder contract

`struct Finding: Decodable, Identifiable, Hashable`
(`APIClient.swift:782-800`). Fields:

```
finding_id, check_id, title, check_title?, description?, severity, status,
resource_arn?, resource_type?, region?, domain,
frameworks: [String: [String]],
remediation?, first_seen, last_seen
```

`id` = `finding_id` (`:799`).

**The load-bearing constraint (CLAUDE.md):** `frameworks` is typed
`[String: [String]]` (`:794`) — a JSON **object** of framework→[control ids],
e.g. `{"soc2": ["CC6.1"], "cis_aws": ["1.2"]}`. If the server sends
`frameworks` as a JSON **array** (`[]`), `JSONDecoder` throws a type-mismatch
and **the whole `Finding` row fails to decode** → the entire
`FindingsResponse` decode fails. This is why backend synthetic findings must
emit `frameworks` as `{}` not `[]`. Same `[String: [String]]` shape on
`FindingGroup.frameworks` (`:584`).

Render sites: `FindingDetailView` (`TopRisksView.swift:396-406`, share text
`:443-449`), `GroupRow`/`GroupDetailView` (`:197-200, 230-238`). All iterate
`frameworks.sorted(by: key)`.

Related: `EntitySummary`/`EntityDetail` (`:804-834`) carry loosely-typed
`AnyJSON` (`:838-880`) for `attributes`/`evidence_packet`, rendered raw via
`prettyJSON()` — these tolerate arbitrary shapes (unlike `Finding`).

## Over-ported web features (vs the companion vision)

The MEMORY/CLAUDE vision is "lightweight alerting + team-handoff companion,
not a web-feature port." Against that, these screens look over-ported:

- **`RisksRegisterView`** (`Views/Register/RisksRegisterView.swift`) — full
  risk-register CRUD with a 6-state status picker, owner/due-date form, and a
  `NewRiskSheet`. Header comment even says "Mirrors web /risks route" (`:4`).
  Heavy authoring surface for a phone companion.
- **`TopRisksView` rollup browser** (`Views/Findings/TopRisksView.swift`) —
  debounced search, client-side framework filtering, domain grouping,
  drill-into-resources. A full findings explorer rather than an alert feed.
- **`OverviewView` charts** (`Views/Overview/OverviewView.swift`) — Swift
  Charts donut + bar + compliance-posture rows replicate the web dashboard.
- The **team-handoff** part of the vision is *under*-built: the only sharing
  primitive (`ShareLink` in `FindingDetailView`) is in dead code (see above).
  `BriefingView` (push→voice briefing) is the clearest expression of the
  companion vision and IS wired.

## Top 3 optimization opportunities

1. **`OverviewView.load()` fires 7 requests, 3 of them redundant**
   (`Views/Overview/OverviewView.swift:131-148`): it calls both
   `findingsTotal()` (count-only) AND `findingsSummary()` (which already
   returns `total`), and `eventsTotal(kind:alert)` AND `listEvents(limit:5)`.
   `findingsSummary.total` could drop the separate `findingsTotal` call;
   the recent-events list could expose its own total. ~2 fewer round-trips on
   every Brief load + pull-to-refresh.
2. **`FindingsResponse.total` is fetched via `limit=1` probe requests**
   (`APIClient.swift:171-187, 209-225` — `findingsTotal`/`eventsTotal` send
   `limit=1` purely to read `total`). The list responses already carry
   `total`; the stat cards could reuse the `limit=5` list call's `total`
   instead of issuing a second request per stat.
3. **`VoiceClient` route-change observer is never removed**
   (`VoiceClient.swift:145-154`): `NotificationCenter.addObserver(forName:
   routeChangeNotification...)` adds a closure observer on every
   `configureRtcAudioSession()` (i.e., every `start()`) but `teardown()`
   (`:239-250`) never removes it. Repeated start/stop cycles accumulate
   observers that keep re-forcing the speaker route. Capture the token and
   remove it in `teardown()`.

## Gotchas / non-obvious behavior

- **ID token, not access token, is the API JWT** (`AuthManager.swift:8-11,
  101`). `access_token` is stored but unused. Any new authed call must go
  through `attachAuthHeader` which uses `validIdToken()`.
- **Cognito config is hardcoded in the binary** (`AuthManager.swift:17-48`),
  but `API_BASE_URL` comes from Info.plist/`Local.xcconfig`
  (`APIClient.swift:8-16`) — two different config mechanisms. Missing
  `API_BASE_URL` is a hard `fatalError` at first access.
- **Per-tenant IdP routing happens before sign-in:** `SignInView` →
  `discoverTenant` (unauthed) returns an `authorize_url` already carrying the
  `identity_provider` hint, so the Cognito Hosted UI skips the picker
  (`APIClient.swift:112-135`, `AuthManager.swift:71-87`).
- **`frameworks` MUST be a JSON object** or the whole `Finding`/`FindingGroup`
  decode fails (see Finding decoder contract).
- **Voice talks directly to OpenAI**, not the backend — only the ephemeral
  key comes from `POST /voice/session` (`VoiceClient.swift:8-13, 210-218`).
  The app's "iOS/web never call upstream" rule is intentionally relaxed here
  because the ephemeral key is the rate-limit boundary.
- **`AppState.Tab` has a `.register` case AND `RisksRegisterView`** is the
  3rd tab (`MainTabView.swift:7-11, 28-30`) — "Risks" (TopRisksView) and
  "Register" (RisksRegisterView) are two distinct tabs, easy to conflate.
- **Two `VoiceClient` instances exist concurrently is avoided** because
  `BriefingView` is a modal sheet and `VoiceChatView` is its own sheet;
  each owns its own client and stops it `onDisappear`. There is no shared
  singleton.
- **GCP onboarding is wired in `APIClient` but the UI tile is disabled**
  ("Coming in Phase D", `ConnectCloudsView.swift:41`).
